# 语法
## 一. 虚函数
```c++
//这种是纯虚函数  
virtual void init()=0;
```
- 有纯虚函数的类就是抽象类，不能有自己的实例
```c++
class base{  
public:  
    //这种是纯虚函数，没有函数体
    virtual void init()=0;
    //虚函数可以有函数体  
    virtual void init(){}
};
  
class A:public base{  
public:  
    //实现虚函数  
    void init(){  
        cout<<"A"<<endl;  
    }  
};
```
多态下虚函数的本质就是，父类包含了vfptr指针，指向了vftable，这张表中记录了这个函数的真正访问入口地址，当子类继承父类时会将vfptr指针和表中的入口一起继承下来，当在多态情况下访问函数时，会根据子类的vfptr指针找到子类对应的vftable表，然后找函数的入口，如果子类重写或者实现了虚函数，表中的入口地址就会改变

## 三.继承
父类的所有成员都能继承下来，只是访问级别的问题
- public继承，父类的private访问不到，public成员继承下来还是public，protected还是protected
- protected继承，private访问不到，public和protected变成protected
- private，privat访问不到，public和protected变成protected
创建出子类对象时，父类对象也会创建出来，所以即使父类是抽象类，它的析构函数是纯虚构函数，也必须要有它自己的类外实现
创建子类对象时，默认调用的是父类的无参构造，如果父类只有有参构造，而且子类的构造器没有显式调用父类的有参构造就会找不到父类的构造定义
# 智能指针
	相较于*表示的裸指针，智能指针在其工作区结束后会自动释放内存，智能指针本身是一个对象
## 一. 独占指针（unique_ptr）
```c++
class test {
public:
	test() { cout << "test()构造函数被调用" << endl; }
	~test() { cout << "~test()析构函数被调用" << endl; }
	void display() { cout << "display成员函数被调用" << endl; }
};
//形参要求传入独占指针（值传递）
void func(unique_ptr<test> p) {
	p->display();
}
//这种是把独占指针这个对象的引用传过去了
void func(unique_ptr<test> & p) {
	p.display();
}
//这个是把独占指针本身的指针传过去了
void func(unique_ptr<test>  p) {
	（*p）->display();
}
//传入const指针借用资源，但不能修改，符合设计理念
void func(const test* t) {
	t->display();
}
int main() {
	unique_ptr<test> u_p = make_unique<test>();
	//func(u_p);编译会报错
	//转移所有权
	func(move(u_p));
	//借用，get（）获取指向的内存的地址
	func(u_p.get());
}
```
- 独占指针即表示这一块的内存只能由一个指针来管理
- 如果函数的形参是一个独占指针，如果允许拷贝，那么实参和形参都可以管理这个内存违反了设计
- 所以独占指针传递，只能转移所有权或借用资源（但是不能修改）
	- 当转移所有权给形参后，原来的独占指针就不能再用了，由于智能指针在自己的工作区结束后会自动释放内存，所以函数结束后，指针指向的内存就释放了
## 二. 共享指针（share_ptr）和弱引用指针（weak_ptr）
```c++
class test {
public:
	test() { cout << "test()构造函数被调用" << endl; }
	~test() { cout << "~test()析构函数被调用" << endl; }
	void display() const { cout << "display成员函数被调用" << endl; }
	weak_ptr<test> m_friend;
	void set_fridnet(shared_ptr<test> p) {
		this->m_friend = p;
	}
};
//拷贝值传递，发生拷贝会让计数器+1，函数结束形参共享指针释放，计数器-1
void fun(shared_ptr<test> p) {
	p->display();
	//查看计数器
	cout << p.use_count() << endl;
}
//传递的是实参共享指针的引用，相当于取别名，并不是拷贝，计数器不会+1
void fun(shared_ptr<test>& p) {
	p->display();
}
//独占指针可以强转成共享指针
shared_ptr<test> get_ptr() {
	return make_unique<test>();
}
int main() {
	shared_ptr<test> p = make_shared<test>();
	//函数返回独占指针可以强转成共享指针
	shared_ptr<test> p1 = get_ptr();
	//或者通过move转移所有权给共享指针
	shared_ptr<test> p2 = move(make_unique<test>());
	p1->m_friend = make_shared<test>();
}
```
- 共享指针允许值传递和引用传递
- 共享指针对象内部维护了一个计数器，每拷贝一次计数器+1，每销毁一个同源的共享指针计数器-1，当计数器减为0或者整个工作区结束才会将内存释放
- 函数形参是值传递才会发生拷贝，计数器+1，引用传递相当于取别名不属于拷贝，所以计数器不变
- 如果两个对象分别保存对方的共享指针，在销毁时会出现循环依赖问题（在销毁自己时需要销毁对方，对方反过来也要销毁自己），导致程序崩溃，通过弱引用指针，只做标记作用，不能正常使用弱引用指针，没有所有权，可以解决循环依赖问题
